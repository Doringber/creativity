# הוספת דמויות (Quirky Series וכו') למשחק

המשחק הוא **2D**, אז כל דמות צריכה להיות **PNG עם רקע שקוף**.
יש לך מודלי **FBX** — אין בעיה, יש שתי דרכים להכניס אותם:

---

## דרך א׳ — אם בחבילה כבר יש תמונות PNG (הכי מהיר ⚡)
חבילות כמו Quirky Series כוללות לעיתים תיקיית **`Thumbnails` / `Icons` / `Renders`**
עם תמונות מוכנות של כל דמות.
1. חפש תיקייה כזו בחבילה.
2. העלה את ה-PNG-ים ישירות ל-`assets/sprites/` (ראה "איך מעלים" למטה).
3. כתוב לי "העליתי" — ואחבר אותם למשחק.

## דרך ב׳ — יש לך רק FBX (אני ממיר אוטומטית 🤖)
כבר בדקתי שזה עובד אצלי: אני ממיר כל **FBX → GLB → PNG שקוף** ומחבר אותם.
1. העלה את קובצי ה-**`.fbx`** (ואם יש — תיקיות `Textures/` לידם) אל **`assets/source/`**.
2. כתוב לי "העליתי".
3. אני מריץ את ההמרה, מייצר ספרייטים ב-`assets/sprites/`, ומחבר אותם כדמויות שאפשר לתפוס.

> אם תוכל להמיר ל-**GLB** לפני ההעלאה (למשל ב-Blender: Import FBX → Export glTF Binary) — עוד יותר טוב, אבל לא חובה.

---

## איך מעלים קבצים ל-repo (בלי git)
1. ב-GitHub, היכנס ל-repo `Doringber/creativity` ובחר את ה-branch `claude/pokemon-go-ar-game-ocWUA`.
2. כנס לתיקייה (`assets/source` או `assets/sprites`).
3. **Add file → Upload files** → גרור את הקבצים → **Commit**.
   - אפשר לגרור כמה קבצים יחד. מגבלה ~25MB לקובץ בממשק הווב.
   - **לחלץ את ה-ZIP קודם** ולהעלות את הקבצים עצמם (לא את ה-ZIP).

---

## המלצות לתוצאה הכי טובה
- **בחר ~8–12 דמויות** (לא חובה את כל ה-30+) — משחק וובי צריך להישאר קליל ומהיר לטעינה.
- שמות קבצים באנגלית, פשוטים: `dragon.fbx`, `cactoro.fbx`, `yeti.fbx`...
- אני אקבע רמת נדירות לכל דמות (רגיל/נדיר/אגדי) — או תגיד לי העדפה.
- רישוי: שימוש פנימי/אישי בחבילה שרכשת — תקין.

## איך זה ייראה אחרי שאחבר
כל דמות תהפוך ל"פנגו" שאפשר לתפוס: תופיעה ב-AR, תקפוץ/תזוז, תיכנס ל**פנגודקס**,
ותהיה לה רמת נדירות וניקוד. הדמויות הנדירות יזהרו ויהיו שוות יותר.
